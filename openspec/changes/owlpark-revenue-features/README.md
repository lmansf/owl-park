# owlpark-revenue-features

Add five honest, arithmetic-driven revenue features to the `features/` plugin system — cross-sell,
add-ons, donations, gifting, and membership conversion — plus the small cart-model extension they
need (per-line metadata and non-catalog lines).

- `proposal.md` — why, what changes, capabilities, impact
- `design.md` — the cart line model extension, the honesty rules, and per-feature design
- `specs/cart-line-model/spec.md` — MODIFIED cart/catalog capability deltas
- `specs/revenue-enhancement-modules/spec.md` — ADDED requirements for the five modules
- `tasks.md` — implementation checklist
